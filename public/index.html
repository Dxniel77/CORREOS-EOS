Historial de campañas
Cómo configurar en 3 pasos
1. Crea cuenta gratis en resend.com → API Keys → Create API Key → copia la key
2. Despliega este proyecto en vercel.com → agrega la variable RESEND_API_KEY con tu key
3. Pega la URL de Vercel abajo y haz la prueba
Configuración del servidor
Variables de entorno en Vercel

Ve a tu proyecto en Vercel → Settings → Environment Variables y agrega estas dos:

RESEND_API_KEY     =  re_xxxxxxxxxxxxxxxxxxxx
FROM_EMAIL         =  EOS Community <onboarding@resend.dev>
Nota sobre el email remitente: Con onboarding@resend.dev el email llega desde ese dominio de Resend. Para que salga desde tu propio dominio necesitas verificar un dominio en Resend. Con Gmail sin dominio propio, siempre habrá riesgo de ir a Promociones — es una limitación de Gmail, no del bot.
Cómo subir a GitHub y Vercel

1. Crear repo en GitHub
Ve a github.com → New repository → nombre: mailbot-eos → Public → Create. Luego sube todos los archivos de la carpeta.

2. Conectar Vercel
Ve a vercel.com → Sign up con GitHub → New Project → importa mailbot-eos → Deploy.

3. Agregar API Key
En Vercel → tu proyecto → Settings → Environment Variables → agrega RESEND_API_KEY y FROM_EMAIL → Redeploy.

4. Configurar el bot
Abre tu URL de Vercel → ve a Configuración → pega la URL → prueba.
